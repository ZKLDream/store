# 头像登录/退出登录功能计划

## 仓库研究结论

当前项目已有用户信息存储机制，包括：
- 本地存储用户头像和用户名
- 云开发初始化
- 获取用户 openid 功能

需要添加：
- 点击头像显示操作菜单
- 切换登录（修改头像和用户名）
- 退出登录（恢复默认用户）

## 需要修改的文件

1. `src/store/AppContext.tsx` - 添加登录/退出相关的状态和方法
2. `src/pages/profile/index.tsx` - 添加头像点击交互和操作弹窗
3. `src/pages/profile/index.module.scss` - 添加弹窗样式

## 修改步骤

### 步骤 1: 在 AppContext 中添加登录/退出功能
在 `src/store/AppContext.tsx` 中：
- 添加 `updateUserProfile` 方法 - 更新用户头像和用户名
- 添加 `clearUserProfile` 方法 - 恢复默认用户
- 在 context 中暴露这两个方法

### 步骤 2: 在个人页面添加头像点击和操作弹窗
在 `src/pages/profile/index.tsx` 中：
- 添加头像可点击样式
- 添加 `showUserMenu` state 控制菜单显示
- 添加点击头像显示菜单的逻辑
- 菜单包含两个选项：
  - 编辑个人信息（修改头像和用户名）
  - 退出登录（恢复默认）

### 步骤 3: 添加编辑个人信息弹窗
在个人页面中：
- 添加编辑用户信息的弹窗
- 包含头像选择和用户名输入
- 保存时调用 `updateUserProfile`

### 步骤 4: 添加退出登录功能
在个人页面中：
- 点击退出登录时，调用 `clearUserProfile`
- 显示确认提示

### 步骤 5: 添加弹窗样式
在 `src/pages/profile/index.module.scss` 中：
- 添加操作菜单样式
- 添加编辑个人信息弹窗样式

## 依赖和注意事项

1. 使用微信小程序的 `chooseImage` 选择头像
2. 使用云存储上传头像图片
3. 退出登录时清除本地存储的用户信息
4. 保持与现有代码风格一致

## 风险处理

1. 如果上传头像失败，使用默认头像
2. 退出登录前显示确认提示
3. 确保所有操作都有友好的提示
