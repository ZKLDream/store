# 首页商品数据刷新计划

## 仓库研究结论

当前问题：
1. 首页在 `useEffect(() => { loadFruitsData(); }, [])` 中只在组件挂载时加载一次数据
2. 当用户从商品管理页面编辑/删除商品后返回首页时，首页不会刷新数据
3. 需要在每次页面显示时都重新加载商品数据

## 需要修改的文件

1. `src/pages/home/index.tsx` - 添加页面显示时刷新数据的逻辑

## 修改步骤

### 步骤 1: 添加 useDidShow 钩子
在首页组件中：
- 从 `@tarojs/taro` 导入 `useDidShow`
- 使用 `useDidShow` 钩子，每次页面显示时调用 `loadFruitsData()` 重新加载数据
- 保留原有的 `useEffect` 用于首次加载

## 依赖和注意事项

1. `useDidShow` 会在每次页面显示时触发，包括从其他页面返回时
2. 这样可以确保用户编辑/删除商品后返回首页能看到最新数据
3. 保留原有的 `useEffect` 确保首次加载也能正常工作

## 风险处理

1. 频繁刷新可能影响性能，但对于商品数据这种不频繁变更的数据，影响可以接受
2. 可以考虑后续添加数据缓存机制，但当前需求只需要简单的刷新即可
