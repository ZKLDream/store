# 自定义斤数输入功能实现计划

## 需求分析
- 用户希望在首页点击"加入清单"时弹出输入框
- 支持自定义输入斤数，而不是只能选择预设的规格
- 保持原有的用户体验和视觉设计

## 当前代码结构
1. ProductCard组件：显示水果信息、规格选择、加入清单按钮
2. HomePage：管理分类和清单逻辑
3. AppContext：管理全局状态

## 实现方案

### 1. 修改ProductCard组件
- 移除预设规格选择（1斤、2斤、3斤、5斤）
- 添加输入框弹窗状态管理
- 实现输入框弹窗UI
- 支持自定义输入斤数
- 实时计算价格和成本价

### 2. 修改HomePage组件
- 更新handleAddToList函数，支持自定义斤数
- 调整数据传递逻辑

### 3. 修改AppContext
- 更新addToList函数，支持自定义斤数作为规格
- 确保数据结构兼容

### 4. 添加输入框样式
- 创建美观的输入框弹窗样式
- 确保在不同设备上显示正常

## 文件修改清单
1. `src/components/ProductCard/index.tsx` - 添加输入框弹窗功能
2. `src/components/ProductCard/index.module.scss` - 添加输入框样式
3. `src/pages/home/index.tsx` - 更新逻辑
4. `src/store/AppContext.tsx` - 更新状态管理（如有需要）

## 实现步骤
1. 修改ProductCard组件，添加输入框弹窗状态
2. 创建输入框弹窗UI
3. 实现输入验证和实时计算
4. 更新HomePage的数据传递逻辑
5. 测试功能是否正常工作

## 用户体验优化
- 点击"加入清单"弹出输入框
- 输入框默认值可以设为1
- 支持小数输入（如1.5斤）
- 实时显示总价
- 点击确认后添加到清单
- 点击取消或空白处关闭弹窗

## 风险和注意事项
- 确保数据类型正确（斤数可能是小数）
- 保持现有功能不被破坏
- 输入验证防止非法字符
- 确保在小程序环境中正常显示
