# 水果店记账本小程序 - 产品需求文档

## Overview

- **Summary**: 一款专为水果店设计的记账本小程序，帮助店主记录每日销售情况、计算盈利，支持水果销售清单管理和数据统计。
- **Purpose**: 解决个人小程序无法使用电商功能的问题，提供合规的记账功能，帮助水果店店主更好地管理销售数据和盈利情况。
- **Target Users**: 水果店店主、店员

## Goals

- 实现水果销售清单管理功能
- 支持每日销售数据统计和盈利计算
- 提供个人中心功能
- 确保小程序符合个人主体审核要求

## Non-Goals (Out of Scope)

- 在线支付功能
- 购物车功能（改为清单管理）
- 订单系统（改为销售记录）
- 用户注册登录系统
- 多用户权限管理

## Background & Context

- 原小程序因涉及电商模式（购物车、订单功能）被小程序平台拒审
- 调整为记账本模式，符合个人主体小程序的功能范围
- 保留原有的水果数据和分类功能

## Functional Requirements

- **FR-1**: 水果商品展示功能
  - 显示水果名称、描述、成本价、售价、图片
  - 支持按分类查看水果
  - 支持规格选择（1斤、2斤、3斤、5斤）
- **FR-2**: 清单管理功能
  - 支持将水果加入销售清单
  - 支持调整清单中水果的数量
  - 支持从清单中移除水果
- **FR-3**: 销售统计功能
  - 显示每日销售的水果列表
  - 显示每个水果的销售数量（斤数），并且可以编辑
  - 显示每个水果的成本价和售价，并且可以编辑
  - 计算每个水果的利润，在右侧显示，并且可以编辑
  - 计算每日总销售额和总利润，不可编辑，随单个水果利润刷新
- **FR-4**: 个人中心功能
  - 显示记账本基本信息
  - 提供使用说明
  - 显示版本信息

## Non-Functional Requirements

- **NFR-1**: 性能要求
  - 页面加载时间不超过3秒
  - 操作响应时间不超过1秒
- **NFR-2**: 可用性要求
  - 操作流程简单直观
  - 界面设计清晰易读
  - 支持常见水果数据的快速录入
- **NFR-3**: 数据持久化
  - 使用本地存储保存销售数据
  - 确保数据不丢失

## Constraints

- **Technical**: Taro 4.x 框架，React 18 + TypeScript
- **Business**: 符合个人主体小程序审核要求，不包含电商功能
- **Dependencies**: 本地存储 API

## Assumptions

- 水果数据使用本地静态数据
- 销售记录存储在本地
- 不需要网络连接即可使用

## Acceptance Criteria

### AC-1: 水果商品展示

- **Given**: 进入首页
- **When**: 查看水果列表
- **Then**: 能够看到水果图片、名称、描述、价格等信息
- **Verification**: `human-judgment`

### AC-2: 加入清单功能

- **Given**: 选择水果规格
- **When**: 点击"加入清单"按钮
- **Then**: 水果成功加入清单，清单数量更新
- **Verification**: `programmatic`

### AC-3: 清单管理

- **Given**: 进入清单页面
- **When**: 调整水果数量或移除水果
- **Then**: 清单数据实时更新
- **Verification**: `programmatic`

### AC-4: 销售统计

- **Given**: 在清单页面
- **When**: 查看销售数据
- **Then**: 能够看到每个水果的销售数量、成本价、售价和利润
- **Verification**: `programmatic`

### AC-5: 盈利计算

- **Given**: 在清单页面
- **When**: 点击"结算"按钮
- **Then**: 显示每日总销售额和总利润
- **Verification**: `programmatic`

### AC-6: 个人中心

- **Given**: 进入"我的"页面
- **When**: 查看个人中心内容
- **Then**: 能够看到记账本信息和使用说明
- **Verification**: `human-judgment`
